# AI-Optimized Documentation

## Goal
Create AI-friendly docs that minimize token usage while preserving complete intent and context.

## Tasks
- [ ] Create AI-optimized versions of key docs (README, migration guides, etc.)
- [ ] Add hidden notes in human docs pointing AI to the optimized versions
- [ ] Use compact notation, abbreviations, and structured formats for AI docs
- [ ] Ensure no loss of information despite compression
- [ ] Ensure only AI docs are published with the package, replacing the readme.md

## Format Ideas
- Use tables/lists instead of prose
- Remove redundant examples (keep 1 per concept)
- Use shorthand notation where unambiguous
- Focus on rules, patterns, gotchas vs explanations
