ESM + Bundler

Install the package and use with Vite, esbuild, or Rollup. Full framework with decorators, routing, and controllers.

# Install
npm install snice

# Or scaffold a full project
npx snice create-app my-app
import { element, property, render, html } from 'snice';

@element('my-counter')
class MyCounter extends HTMLElement {
  @property({ type: Number }) count = 0;

  @render()
  template() {
    return html`<button @click=${() => this.count++}>${this.count}</button>`;
  }
}

Best for: Full applications, SPAs, projects using a bundler.

Standalone

Each component is a single file with the Snice runtime bundled in. Drop a <script> tag into any page — no build step, no dependencies.

<!-- Single script tag, works anywhere -->
<script src="snice-button.min.js"></script>
<snice-button variant="primary">Click me</snice-button>

Each bundle is ~20–40KB minified. Since the runtime is included in every file, loading multiple components duplicates it.

Best for: 1–2 components, CMS pages, WordPress, static sites.

Runtime + Light

Load the Snice runtime once, then load lightweight component builds that reference it. Dramatically reduces total size when using multiple components.

<!-- Load runtime once -->
<script src="snice-runtime.min.js"></script>

<!-- Lightweight components -->
<script src="snice-button.light.min.js"></script>
<script src="snice-input.light.min.js"></script>
<script src="snice-modal.light.min.js"></script>
Scenario Standalone Runtime + Light
1 component ~25KB ~20KB
3 components ~75KB ~27KB
10 components ~250KB ~55KB

Best for: 3+ components without a bundler, landing pages, dashboards.

React Wrappers

First-class React adapters with prop mapping, event callbacks, ref forwarding, and form integration. React 17, 18, and 19.

import { Button, Input, Modal } from 'snice/react';

function App() {
  const [email, setEmail] = useState('');

  return (
    <>
      <Input
        label="Email"
        value={email}
        onChange={(e) => setEmail(e.detail.value)}
      />
      <Button variant="primary" onClick={handleSubmit}>
        Submit
      </Button>
    </>
  );
}

Best for: React applications that want Snice UI components with native React DX.

The bare minimum

An element needs a tag name and a template. That's it.

import { element, render, html } from 'snice';

@element('task-item')
class TaskItem extends HTMLElement {
  @render()
  template() {
    return html`<span>Buy groceries</span>`;
  }
}

<task-item></task-item>

Properties

Properties sync with HTML attributes and trigger re-renders automatically.

@element('task-item')
class TaskItem extends HTMLElement {
  @property() label = '';
  @property({ type: Boolean }) done = false;

  @render()
  template() {
    return html`<span>${this.label}</span>`;
  }
}

<task-item label="Buy groceries"></task-item>

Styles

Styles are scoped to the element via shadow DOM. They won't leak out.

@styles()
css() {
  return css`
    :host { display: flex; align-items: center; padding: 0.5rem; }
    :host([done]) span { text-decoration: line-through; opacity: 0.5; }
  `;
}

Use :host to style the element itself. :host([done]) targets when the done attribute is present.

Inline handlers

Use @event=${handler} in templates. Handlers are auto-bound to this.

@render()
template() {
  return html`
    <input type="checkbox"
      .checked=${this.done}
      @change=${this.toggle} />
    <span>${this.label}</span>
  `;
}

toggle() {
  this.done = !this.done;
}

.checked=${val} sets the property (not attribute). ?disabled=${bool} toggles an attribute.

Emitting events

Use @dispatch to fire custom events. The return value becomes event.detail.

@dispatch('status-changed')
toggle() {
  this.done = !this.done;
  return { label: this.label, done: this.done };
}

Parents listen with @status-changed=${handler} in templates, @on('status-changed') as a decorator, or addEventListener('status-changed', ...).

Keyboard shortcuts

Filter events by key or key combo directly in the template.

@render()
template() {
  return html`
    <input
      placeholder="New task..."
      @keydown:Enter=${this.addTask}
      @keydown:Escape=${this.clear} />
  `;
}

Combos: @keydown:ctrl+s, @keydown:shift+Enter. Works on any element.

Event delegation

Handle events declaratively with the @on decorator instead of inline handlers.

@on('click', '.delete')
handleDelete(e) {
  const id = e.target.dataset.id;
  this.items = this.items.filter(i => i.id !== id);
}

@on('input', 'input.search', { debounce: 300 })
handleSearch(e) {
  this.filter = e.target.value;
}

Delegation works on shadow DOM elements. Built-in debounce and throttle options.

Conditionals

Show or hide parts of the template.

@render()
template() {
  return html`
    <input type="checkbox" .checked=${this.done} @change=${this.toggle} />
    <span>${this.label}</span>

    <if ${this.done}>
      <small>Completed!</small>
    </if>
  `;
}

<if> conditionally renders its children. For multi-branch, use <case> / <when>.

Lists

Render arrays with .map(). Use key for efficient updates.

@element('task-list')
class TaskList extends HTMLElement {
  @property({ type: Array }) items = [];

  @render()
  template() {
    return html`
      ${this.items.map(item => html`
        <task-item
          key=${item.id}
          label=${item.label}
          ?done=${item.done}
          @status-changed=${(e) => this.update(e.detail)}>
        </task-item>
      `)}
    `;
  }
}

Watch changes

Run logic when a specific property changes.

@watch('done')
onDoneChanged(oldVal, newVal) {
  console.log(`Task ${this.label}: ${newVal ? 'completed' : 'reopened'}`);
}

Query the DOM

Get references to elements inside your shadow DOM.

@query('input') checkbox!: HTMLInputElement;

@ready()
init() {
  this.checkbox.focus();
}

@ready() runs after the first render, so DOM refs are available.

Snice components export their interfaces from .types.ts, so you can type queries:

import type { SniceInput } from 'snice/components/input';

@query('snice-input') input!: SniceInput;

Ready & Dispose

Setup and teardown.

@ready()
onMount() {
  // Element is rendered and in the DOM
  this.interval = setInterval(() => this.tick(), 1000);
}

@dispose()
onUnmount() {
  // Element is being removed from the DOM
  clearInterval(this.interval);
}

Controllers

Swap behavior on any element without changing its code. A controller attaches logic from the outside.

import { controller, on } from 'snice';

@controller('weather')
class WeatherController {
  async attach(el) {
    // Called when controller binds to an element
    const data = await fetch('/api/weather').then(r => r.json());
    this.city = data.city;
    el.label = data.city;
    el.value = `${data.temp}°`;
  }

  @on('click')
  viewDetails() {
    // Navigate to the city's weather page
    navigate(`/weather/${this.city}`);
  }

  detach(el) {
    // Cleanup when controller is removed
  }
}

Attach by attribute: <stat-card controller="weather"></stat-card>

Swap at runtime: change the attribute to "stocks" and the element gets new behavior, same UI.

Full example

Here's the complete task-item with everything wired together.

import { element, property, watch, query, render, styles,
         dispatch, ready, dispose, html, css } from 'snice';

@element('task-item')
class TaskItem extends HTMLElement {
  @property() label = '';
  @property({ type: Boolean }) done = false;

  @query('input') checkbox!: HTMLInputElement;

  @render()
  template() {
    return html`
      <input type="checkbox" .checked=${this.done} @change=${this.toggle} />
      <span>${this.label}</span>
      <if ${this.done}><small>Done</small></if>
    `;
  }

  @styles()
  css() {
    return css`
      :host { display: flex; align-items: center; gap: 0.5rem; padding: 0.5rem; }
      :host([done]) span { text-decoration: line-through; opacity: 0.5; }
      small { color: green; }
    `;
  }

  @dispatch('status-changed')
  toggle() {
    this.done = !this.done;
    return { label: this.label, done: this.done };
  }

  @watch('done')
  onDone(oldVal, newVal) {
    console.log(this.label, newVal ? 'completed' : 'reopened');
  }
}
